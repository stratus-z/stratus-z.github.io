Thinking 0.2

箭头中的Add the file对应的是Git中的git add命令，用于将未跟踪的文件添加到暂存区，准备提交；Stage the file对应的也是Git中的git add命令，但是是针对已经跟踪的文件进行修改后再次添加到暂存区，准备提交；Commit对应的则是git commit命令，用于将暂存区的内容提交到版本库中，形成一个新的版本。

Thinking 0.3

1、

当代码文件print.c被错误删除时，可以使用git checkout -- print.c命令来恢复被删除的文件。这个命令会从版本库中检出最新版本的print.c文件，并将其恢复到工作区中。

2、

当代码文件print.c被错误删除后还执行了git rm print.c命令，想要恢复可以使用git reset HEAD print.c命令来取消对暂存区print.c文件的删除操作，之后接着使用git checkout -- print.c命令来恢复被删除的文件。

3、

当无关文件hello.txt已经添加到暂存区时，可以使用git restore --staged hello.txt命令来取消对hello.txt文件的暂存操作，使其从暂存区中移除，但不会删除工作区中的文件且不会删除此文件。

在实现过程中，echo echo Shell Start会直接输出echo Shell Start，而不是输出Shell Start，这是因为echo命令会将后面的内容原样输出，但是echo `echo Shell Start`会先执行echo Shell Start命令，得到Shell Start的输出结果，然后再将这个结果作为参数传递给外层的echo命令，最终输出Shell Start，二者效果不同。echo echo $c>file1与echo `echo $c>file1`的效果也不同，前者会将>识别为输出重定向符号，将echo $c输出到file1文件中，而后者会先执行echo $c>file1命令，将$c的值输出到file1文件中，然后再将这个结果作为参数传递给外层的echo命令，最终输出3，故二者效果不同。

2、难点分析

就我个人而言，本次实验的难点主要分为两个方面：Makefile和Shell脚本的编写，Git的使用。

在Makefile和Shell脚本的编写方面，主要难点在于理解和掌握Makefile和Shell脚本的语法和使用方法，以及如何将它们应用到实际的项目中。在这个过程中，需要我们对相关指令的功能和用法有深入的理解，并且需要我们能够灵活地运用这些指令来实现我们的目标，这对于之前没有接触过Linux的我来说有一定的挑战性，需要我不断的去查阅资料，询问大模型，以便我能够正确的使用这些指令来完成实验任务。其中，grep、sed和awk指令的使用比较复杂，可以使用的场景也比较多，需要花比较多的时间和精力才能够熟练的掌握这些指令的使用方法和应用场景。
对于Git的使用，我也非常的不熟练，对于Git管理版本的概念和操作方法还不够熟悉，如思考题中出现的相关版本退回的操作以及需要注意的事项，这些都是我之前没有接触过的内容，需要我通过不断的练习和实践来掌握这些操作方法，并且需要我能够理解这些操作背后的原理和机制，以便我能够更好地使用Git来管理我的代码版本。同时，在pre上机时，由于我的误操作，导致了Git版本库混乱，在助教帮助下才得以恢复。

3、实验体会

通过本次实验，我体会到了Linux系统的强大和灵活性，以及Git作为版本控制工具的重要性和实用性。在实验之前，我完全没有接触过Linux系统，对Git的使用也非常的陌生。但是经过本次实验，我对Linux系统的基本操作有了初步的了解，能够比较熟练的使用实验要求的指令来完成实验任务。这也许可以为我接下来的操作系统实验打下一个比较好的基础，让我能够更好地理解和掌握操作系统的相关知识和技能。在Lab0的上机实验中，通过题目我也感受到了Shell脚本的强大和灵活性，能够通过编写脚本来自动化一些重复性的任务，提高了工作效率，同时也让我体会到了编程的乐趣和成就感。
