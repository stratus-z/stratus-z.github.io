1、思考题

Thinking 3.1

e->env_pgdir[PDX(UVPT)] =PADDR(e->env_pgdir)|PTE_V中UVPT是用户进程页表的起始地址，而PDX(UVPT)是用户进程页表在页目录中的索引，e->env_pgdir[PDX(UVPT)]则是用户进程页表在页目录中的项。通过将用户进程页表的物理地址与PTE_V进行按位或操作，表示该页表项有效，再将设置好有效位的页表项内容写入用户进程页表在页目录中的项即e->env_pgdir[PDX(UVPT)]。所以这句代码的含义是将用户进程页表映射到用户虚拟地址空间中的UVPT位置，使得用户程序可以通过访问UVPT来访问自己的页表。

开启时钟中断

时钟中断在异常处理结束并返回或者进行进程切换并弹出进程上下文时被重新开启。具体来说，当异常处理完，handle_tlb、handle_mod等函数调用完毕后，会直接跳转到ret_from_exception函数，而handle_int函数在处理完时钟中断后会调用schedule函数进行进程调度，然后在schedule函数中会调用env_run函数，在env_run函数中又会调用不返回的env_pop_tf函数，在env_asm.S文件中，env_pop_tf函数会使用RESET_KCLOCK重置时钟中断，然后跳转到ret_from_exception函数。在ret_from_exception函数中，会使用RESTORE_ALL宏恢复所有寄存器的值，也包括状态寄存器CP0_STATUS，开启了中断使能位，最后再使用eret指令返回到用户态，完成时钟中断的重新开启。

Thinking 3.7

在操作系统启动时，在init.c文件中的mips_init函数会调用schedule(0)函数，从而调用了RESET_KCLOCK宏来设置时钟中断的周期，然后开启时钟中断。一旦时钟中断被开启，系统就会定期触发时钟中断，触发中断后系统会将PC寄存器设置为异常处理程序的入口地址，从而进入异常处理流程，跳转到exc_gen_entry函数，进入异常处理程序的入口，然后经过分发到具体的异常处理函数handle_int，在handle_int函数中会调用schedule函数进行进程调度，完成时钟中断的处理。

根据schedule函数中的yield || count == 0 || e == NULL || e->env_status != ENV_RUNNABLE，系统切换进程的情况只有四种：

  1. yield：当当前进程主动调用schedule函数时，传入的参数yield为真，表示当前进程主动放弃CPU，进入调度状态。
  2. count == 0：当当前进程的时间片用完时，count会被设置为0，此时需要进行调度，切换到下一个可运行的进程。
  3. e == NULL：当没有可运行的进程时，e会被设置为NULL，此时需要进行调度，切换到下一个可运行的进程。
  4. e->env_status != ENV_RUNNABLE：当下一个可运行的进程状态不是可运行状态时，需要进行调度，切换到下一个可运行的进程。

切换进程时，如果当前有可运行进程（e != NULL）且状态为可运行（e->env_status == ENV_RUNNABLE），则需要先将该进程从env_sched_list的头部移除然后放到链表的尾部。然后从env_sched_list的头部取出下一个可运行的进程e，如果没有可运行的进程，则使用panic函数报错，否则将时间片长度设置为进程e的优先级，最后将时间片长度减一，并调用env_run(e)函数切换到进程e。

2、难点分析

进程管理

在这次实验中，实现的主要就是对进程的管理，包括进程的创建、调度和销毁等功能。要清晰的理解进程的这些操作的实现细节，并且正确地实现这些功能是比较有挑战性的。在进程运行时，env_run函数需要正确地切换到新的进程，并且在切换过程中需要保存当前进程的状态，以便下次调度时能够正确地恢复进程的状态。这些操作需要对操作系统的底层机制有较深入的理解。

异常处理

Lab3中还剩下很大一部分便是异常处理的相关内容了。这次要实现的异常情况并不是很多，但在设计异常处理函数时，需要我们对异常处理的流程有一个清晰的理解，并且要正确地处理各种异常情况，确保系统的稳定性和安全性。对于课上实验引入的新的异常情况，需要我们深入理解各种数值的含义和操作系统的内存结构，根据处理需求正确的设计异常处理函数的实现细节。

3、实验体会

通过这次实验，我对操作系统的进程管理和异常处理有了更深入的理解。在实现进程管理的过程中，我学会了如何创建、调度和销毁进程，并且理解了进程切换的机制。在实现异常处理的过程中，我学会了如何设计和实现异常处理函数，并且理解了异常处理的流程和各种异常情况的处理方法。在初次了解实验内容时，我以为这会和CO的P7一样比较抽象，但经过认真阅读和理解代码后，我真正了解了这些流程和实现细节，感受到了操作系统的复杂性和精妙之处。
